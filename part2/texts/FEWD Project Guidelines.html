FEWD Project Assessment Guidelines

Overview:

This document outlines the criteria for assessing student projects for Thinkful's Front End Web Development course. For each project, there are criteria that must be met in order for the project to be assigned a "Pass". For some projects, there are additional criteria for an "Honors Pass".

Project 1: Google Clone

To receive a pass, the student project must meet ALL of the following criteria (keeping in mind that the comparison point is the standard Google search page):

  • Page contains the following elements: navbar, Google logo, search bar, and Google Search and I'm Feeling Lucky buttons
  • External style sheet is used for styling
  • Elements in navbar are properly aligned to the left
  • All elements are correctly positioned
  • When browser is resized, elements on page move and/or resize in an acceptable way

Project 4: Hot or Cold

To receive a pass, the student project must meet ALL of the following criteria:

  • When browser is resized, elements on page move and/or resize in an acceptable way
  • The app meets the project requirements: it picks a number at random, prompts user to input a guess, tells user guess is too high too low or correct, and each of these features works as expected.
  • When browser is resized, elements on page move and/or resize in an acceptable way
  • Uses html interface to get user inputs and uses html to provide feedback about user guesses
  • When user inputs a non-numeric value as a guess, app should provide feedback that user needs to supply appropriate input
  • The app should let you start a new game (via a button or link) without making additional calls to the server (e.g., button should not simply refresh the entire page).

For an Honors Pass, the project must also:

  • Provide user with feedback about whether most recent guess is closer(“hotter”) or further (“colder”) than previous guess.
  • The overall user experience for the app should be excellent. For an example of a student project that would meet the “excellent user experience” criteria, see this student project.
